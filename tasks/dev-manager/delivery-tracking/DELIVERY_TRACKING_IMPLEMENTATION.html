2. API Endpoints

Driver Endpoints (/api/drivers)

File: /home/silverring/projects/printke/src/api/drivers.py

POST /api/drivers/login

Driver login with phone number and password. Returns JWT token.

GET /api/drivers/me

Get current driver information (requires authentication).

GET /api/drivers/deliveries

Get driver's assigned deliveries (assigned and in_transit).

POST /api/drivers/deliveries/{id}/start

Start a delivery (marks as in_transit, records start time).

POST /api/drivers/deliveries/{id}/location

Update GPS location during delivery (saves to location history).

POST /api/drivers/deliveries/{id}/complete

Complete delivery (marks as delivered, optional proof).

GET /api/drivers/deliveries/{id}

Get detailed delivery information including order details.

Update driver information (name, phone, vehicle, status).

POST /api/admin/orders/{order_number}/assign

Assign an order to a driver for delivery.

GET /api/admin/deliveries/active

Get all active deliveries with real-time driver locations.

3. WebSocket Real-Time Tracking

File: /home/silverring/projects/printke/src/api/websockets.py

WS /api/ws/deliveries

WebSocket endpoint for real-time delivery tracking.

Features

  • Customer View: Connect with order_number to track specific delivery
  • Admin Dashboard: Connect without order_number to receive all updates
  • Current Status: Immediately receives current delivery status on connection
  • Live Updates: Real-time location updates as driver moves
  • Status Changes: Broadcasts when delivery status changes
  • Connection Management: Automatic cleanup of dead connections

Message Types

Message Type Description When Sent
connected Connection confirmation On WebSocket connect
current_status Current delivery status After connection (if delivery exists)
location_update Driver location update When driver updates location
status_update Delivery status change When delivery status changes
pong Keep-alive response Response to client ping

5. Usage Examples

Create a Driver

curl -X POST http://localhost:8000/api/admin/drivers \ -H "Authorization: Bearer {admin_token}" \ -H "Content-Type: application/json" \ -d '{ "name": "John Doe", "phone": "+254712345678", "password": "driver123", "vehicle_type": "Motorcycle", "vehicle_plate": "KAA 123X" }'

Assign Order to Driver

curl -X POST http://localhost:8000/api/admin/orders/PK-251231-5067/assign \ -H "Authorization: Bearer {admin_token}" \ -H "Content-Type: application/json" \ -d '{"driver_id": 1}'

Driver Login

curl -X POST http://localhost:8000/api/drivers/login \ -H "Content-Type: application/json" \ -d '{ "phone": "+254712345678", "password": "driver123" }'

Start Delivery

curl -X POST http://localhost:8000/api/drivers/deliveries/1/start \ -H "Authorization: Bearer {driver_token}"

Update Location

curl -X POST http://localhost:8000/api/drivers/deliveries/1/location \ -H "Authorization: Bearer {driver_token}" \ -H "Content-Type: application/json" \ -d '{ "lat": -1.2921, "lng": 36.8219, "accuracy": 10.0, "speed": 30.0 }'

WebSocket Connection (JavaScript)

const ws = new WebSocket('ws://localhost:8000/api/ws/deliveries?order_number=PK-251231-5067'); ws.onmessage = (event) => { const data = JSON.parse(event.data); if (data.type === 'location_update') { console.log(`Driver location: ${data.lat}, ${data.lng}`); // Update map marker } if (data.type === 'status_update') { console.log(`Delivery status: ${data.status}`); // Update UI } };

6. Technical Implementation

Authentication

  • Separate JWT authentication for drivers and admins
  • Driver tokens include driver_id in payload
  • Admin tokens use existing admin authentication
  • Password hashing using bcrypt

Database

  • SQLAlchemy async ORM
  • Automatic table creation via init_db()
  • Proper foreign key relationships
  • Indexed columns for performance (phone, status, timestamp)

Real-Time Features

  • WebSocket connection manager for multiple clients
  • Separate channels for admin and customer views
  • Automatic dead connection cleanup
  • Broadcast to all or specific order watchers

Error Handling

  • Comprehensive HTTP status codes
  • Validation using Pydantic schemas
  • Business logic validation (e.g., can't delete driver with active deliveries)
  • Logging for all major operations
